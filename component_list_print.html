Data Layer — Docker Compose (same Droplet) FREE — SELF-HOSTED
ComponentPort / ProtocolPurpose
PostgreSQL 16:5432 TCPPrimary relational database. Tables: devices, homes, communities, home_community (many-to-many), broadcasts, broadcast_acks, firmware_versions, ota_events, audit_log, users. Docker volume: /pgdata.
Mosquitto MQTT:8883 TLSMessage broker with TLS + ACL authentication. Topics: fleet/{device}/heartbeat (30s interval), fleet/{device}/register (on boot), fleet/broadcast/community/{group_id} (server → homes fan-out), fleet/broadcast/home/{home_id} (direct), fleet/{home_id}/broadcast/ack (delivery receipt), fleet/{device}/ota/cmd + status.
WireGuard VPN:51820 UDPVPN server — encrypted tunnel from each patient home to Droplet. Bypasses Cloudflare (direct IP). Carries MQTT traffic and syslog UDP. Each home gets a unique peer config with pre-shared key.
Monitoring Layer — Docker Compose (same Droplet) FREE — OPEN SOURCE
ComponentPort / ProtocolPurpose
Grafana OSS:3000 HTTPMonitoring and visualization dashboard. Datasources: Loki (logs), PostgreSQL (fleet metrics). Dashboards: device health heatmap, fleet uptime %, OTA rollout tracker, broadcast delivery rates, RSSI signal map. Alerting via email/Slack.
Loki:3100 HTTPLog aggregation engine. Indexes and stores device syslog data from Rsyslog. Queryable via LogQL in Grafana. Low resource — indexes labels only, stores log lines compressed.
Rsyslog:514 UDPSyslog collector. Receives ESP32 device logs over UDP via WireGuard VPN. Parses, tags by device name, forwards structured logs to Loki for indexing and search.

Background Services — Inside FastAPI Process ALWAYS RUNNING
ServiceTriggerPurpose
Heartbeat MonitorEvery 30sProcesses MQTT heartbeat messages (uptime, free heap, RSSI). Updates device status in PostgreSQL. Marks device dead after 90s silence. Triggers Alert Service.
Device RegistrarOn MQTT registerAuto-creates new devices in PostgreSQL when ESP32 boots and publishes MAC/version/role. Assigns device to home and community group.
Broadcast SchedulerCron + on-demandFires scheduled broadcasts at configured times. Enforces Do-Not-Disturb hours per home. Retries undelivered messages for homes that were offline.
Alert ServiceOn dead deviceSends alerts when critical devices go silent — email (SendGrid), Slack webhook, Home Assistant notification. Configurable per device criticality.

ESP32 Devices — ESPHome Firmware (per device) N DEVICES PER HOME
ComponentInterval / TriggerPurpose
ESPHome FirmwareAlways runningYAML-configured firmware. Handles WiFi, MQTT, OTA, sensors, heartbeat. No C++ coding — declarative config, version-controlled in Git.
MQTT HeartbeatEvery 30sPublishes to fleet/{device}/heartbeat — JSON payload: uptime (seconds), free heap (bytes), WiFi RSSI (dBm).
MQTT RegisterOn bootPublishes to fleet/{device}/register — MAC address, firmware version, device role.
OTA ClientOn MQTT commandReceives OTA command via MQTT. Downloads .bin from Cloudflare CDN URL. Validates checksum, flashes, reboots. ACKs success/failure.
Syslog ClientOn log eventSends device logs via UDP syslog to fleet server Rsyslog (:514) through WireGuard VPN.
SensorsVariesFall detection, emergency button, motion sensor, door sensor — varies per home and patient needs. Reports to Home Assistant via ESPHome API.
